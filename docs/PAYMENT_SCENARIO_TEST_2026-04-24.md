## 판단

### 통과한 영역

- 기존 구매 상품 다운로드
- 추가 다운로드
- 로그아웃/재로그인 후 다운로드 권한 유지
- 무료 상품 즉시 다운로드
- 비로그인 접근 제어

### 오픈 전 치명 이슈

1. 카드 결제 진입 실패
   카드 결제 체크아웃에서 토스 위젯 완료 상태와 결제 CTA를 확인하지 못했다.

2. 무통장 입금 신청 실패
   버튼은 노출되지만 실제 주문 상태 전환이나 안내 페이지 이동이 발생하지 않았다.

## 현재 결론

다운로드/권한/무료 전환 시나리오는 실사용 기준으로 통과했다.

하지만 신규 유료 결제 2축이 모두 통과하지 못했기 때문에, 현재 상태를 `결제 오픈 가능`으로 판단할 수 없다.

## 즉시 권장 조치

1. 체크아웃 초기화 단계에서 `orderId`, `dbOrderId`, `ready`, `loading`, `widgets.renderPaymentMethods()` 예외를 서버/클라이언트 로그로 남긴다.
2. `NEXT_PUBLIC_TOSS_CLIENT_KEY`, 토스 결제위젯 어드민 설정, `variantKey=DEFAULT` 유효성을 실운영 값으로 재확인한다.
3. `/api/payment/bank-transfer` 호출이 실제로 발생하는지 네트워크 로그와 서버 로그를 동시에 확인한다.
4. 기존 pending 주문 재사용 정책이 신규 체크아웃을 막고 있지 않은지 점검한다.
5. 위 두 결제 시나리오가 실결제로 PASS 되기 전에는 외부 오픈을 보류한다.
