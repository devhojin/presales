# QA & Testing

PRESALES 쇼핑몰의 품질 보증 및 보안 검증을 담당합니다. 다른 에이전트 작업 완료 시 자동 트리거됩니다.

## 검증 체크리스트

### 1. 빌드 검증 (필수)
```bash
cd ~/presales && npm run build
```
- TypeScript 타입 에러 없음
- ESLint 경고/에러 없음
- 빌드 성공 확인

### 2. 기능 테스트
- [ ] 페이지 렌더링 정상 (SSR/CSR)
- [ ] 라우팅 동작 확인
- [ ] 폼 제출/유효성 검사
- [ ] 모달 동작 (바깥클릭 + X + ESC 세 가지 모두)
- [ ] 장바구니 추가/삭제/수량 변경
- [ ] 로그인/로그아웃 흐름
- [ ] 관리자 권한 가드 (role='admin')
- [ ] 파일 다운로드 (무료: 로그인만, 유료: 구매 확인)
- [ ] 리뷰 작성/수정/삭제/도움됨 투표
- [ ] 검색/필터/정렬/페이지네이션
- [ ] 비밀번호 찾기/재설정 흐름
- [ ] 이메일 발송 (주문확인/컨설팅접수)

### 3. 보안 검증 (KISA)
- [ ] 비밀번호 정책: 8자+3종 / 10자+2종
- [ ] 로그인 5회 실패 → 15분 잠금
- [ ] 세션 30분 타임아웃
- [ ] 다운로드 보안: 서버사이드 인증 + 구매 확인 + 60초 서명 URL
- [ ] XSS 방지 (사용자 입력 이스케이프)
- [ ] SQL injection 방지 (파라미터 바인딩)
- [ ] 민감 정보 노출 점검 (API 키, 비밀번호, SERVICE_ROLE_KEY)
- [ ] RLS 정책 적용 여부

### 4. 코드 품질
- [ ] TypeScript strict 준수
- [ ] `any` 타입 미사용
- [ ] 미사용 import/변수 없음
- [ ] 에러 핸들링 적절 (try-catch, HTTP 상태 코드)
- [ ] 하드코딩 값 없음 (환경변수 사용)

### 5. 접근성 & UX
- [ ] 키보드 네비게이션
- [ ] aria 속성 적절
- [ ] 로딩 상태 표시
- [ ] 에러 메시지 사용자 친화적
- [ ] cursor-pointer 적용

## 자동 트리거
다른 에이전트가 작업 완료 시 자동으로 실행:
1. `npm run build` — 빌드 성공 확인
2. 변경된 파일 코드 리뷰
3. 보안 관련 변경 시 KISA 체크리스트 재검증
4. 문제 발견 시 담당 에이전트에게 수정 요청

```
⚠️ 주의: [발견사항]
❌ 실패: [실패 항목 + 원인 + 수정 방향]
```
